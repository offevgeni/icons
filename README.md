# My Icons Studio

Веб-инструмент на SvelteKit для ведения личной библиотеки SVG-иконок:
- загрузка SVG через браузер;
- автоматическая генерация Svelte-компонентов;
- просмотр/поиск/удаление иконок;
- one-click публикация изменений в Git (commit + push);
- подключение библиотеки в другие Svelte-проекты.

## 1. Быстрый старт

```bash
npm install
npm run dev
```

Откройте `http://localhost:5173`.

## 2. Как подготавливать иконки в Adobe Illustrator

1. Нарисуйте иконку на артборде.
2. Экспортируйте в SVG (`File -> Export -> Export As... -> SVG`).
3. Рекомендуется:
1. `Styling: Presentation Attributes`.
2. `Font: Convert to Outlines`.
3. `Responsive: Off` (опционально).
4. Назовите файл осмысленно (например, `arrow-left.svg`) - имя конвертируется в компонент (`ArrowLeft`).

## 3. Работа через веб-интерфейс

### Генерация

1. В блоке загрузки выберите SVG-файлы.
2. Выберите режим:
1. `Monochrome (currentColor)` - цвета приводятся к `currentColor`.
2. `Original colors` - сохраняются исходные цвета.
3. Нажмите `Сгенерировать`.

Что происходит автоматически:
- SVG оптимизируются (SVGO);
- небезопасные элементы/атрибуты чистятся;
- `id` в SVG префиксуются, чтобы не конфликтовали градиенты/маски;
- генерируются файлы:
1. `src/lib/icons/<IconName>.svelte`
2. `src/lib/icons/__raw/<IconName>.svg`
3. `src/lib/icons/index.js` (переэкспорт)

### Управление библиотекой

- Поиск по имени иконки.
- Кнопка `Копировать импорт`.
- Удаление одной иконки.
- `Очистить всё` для полной очистки библиотеки.

## 4. One-click Git публикация

В блоке `Git публикация`:

1. Нажмите `Обновить статус` (ветка, upstream, изменения).
2. При необходимости укажите `Commit message`.
3. Оставьте включенным `Сразу сделать push после commit`.
4. Нажмите `Опубликовать в Git`.

Инструмент сделает:
1. `git add -A -- src/lib/icons`
2. `git commit -m "..." -- src/lib/icons`
3. `git push` (если включено)

Важно:
- В commit попадают только изменения библиотеки иконок в `src/lib/icons`.
- Если upstream для ветки еще не настроен, UI подскажет. Один раз выполните в терминале:

```bash
git push --set-upstream origin <ваша-ветка>
```

После этого push из UI будет работать по одной кнопке.

## 5. Подключение библиотеки в другой Svelte-проект

### Установка из Git-репозитория

В другом проекте:

```bash
npm i git+ssh://git@github.com/<you>/<repo>.git#main
```

Если используете другую ветку, замените `#main`.

### Импорт

```svelte
<script>
  import { ArrowLeft } from '@offevgeni/icons';
</script>

<ArrowLeft size={20} class="text-slate-700" />
```

Компоненты поддерживают:
- `size` (по умолчанию `24`)
- `title` (для accessibility)
- остальные SVG-атрибуты через rest props (`class`, `style`, `stroke-width`, и т.д.)

## 6. Структура

- `src/routes/+page.svelte` - веб-интерфейс управления.
- `src/routes/api/icons/+server.js` - API генерации/удаления/листа.
- `src/routes/api/git/+server.js` - API git-статуса и публикации.
- `src/lib/server/icon-generator.js` - генерация и оптимизация иконок.
- `src/lib/server/git-publisher.js` - безопасная публикация в git только для иконок.

## 7. Полезно знать

- Для приватных репозиториев убедитесь, что на машине настроен доступ (`ssh-agent`, ключи, токены).
- UI для Git-публикации рассчитан на локальный trusted-режим (ваш личный инструмент).
